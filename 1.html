<u > , < /u > = 밑줄)

6. 줄바꿈

< br> : 줄바꿈 코드 (닫기 필요 X) : 72%

장점: 의미론적으로 뛰어남 (어디부터 어디까지인지 표현이 가능) 81.5% < p styple="margin-top:40px;"> : 여백 위를 40px떨어트림 [css문법! 풍부히 재현가능]

7. HTML이 중요한 이유

폰트를 바꾼것(일반인)과 태그 작성(지식인)의 차이

웹의 철학
1. 모든 운영체제에서 동작한다
2. 모두가 볼 수 있다.
3. 저작권이 없는 공공재이다.
=>Accessibility

9. 부모 자식과 목록

부모와 자식 태그

: < Par > < Chi >< / Chi >< /Par> 와 같은 형식 (무조건으로 이러한 관계임)
< li > , < /li > : 목차(목록) 태그 [자식]
+< ul > < /ul > : 리스트들의 경계를 구분{Unordered list}
+< ol > < /ol > : + 1. 2. 3.등을 자동으로 붙임 {Ordered list}
팁 : ctrl누르고 클릭하면 커서가 복제된다.

10. 문서구조와 슈퍼스타들

< title > , < /title > : 페이지의 제목을 명시함 (검색 시 잘 뜨게 함)
< meta charset="utf-8"> , < / > : 한글 출력을 위한 태그 (utf-8 : 한글 출력을 가능케하는 형식)
< body> , < /body > : 본문을 묶는 태그
< head> , < /head > : 머리말을 묶는 태그
< html> , < /html > : 모두를 묶는 태그
< !doctype html> : html로 작성했음을 나타내는 관용적 표현 태그

11. HTML 태그의 제왕

< a> , < /a> : 링크를 위한 코드 [가장 중요함] {Anchor의 약자}
+ href="주소" : 주소 입력 (hypertext reference)
+ target: "_blank" : 새탭에 열기
+ title="설명" : 마우스 갖다대면 나오는 설명

웹서버와 HTTP


내 컴퓨터가 Web Browser, Web Server역할을 동시에 함.
127.0.0.1/index.html : index.html 파일을 화면에 출력
htdocs(hypertext documents)에 있는 index 파일.
여기에 작업한 파일들을 업로드 해줘야함.

127.0.0.1(Ip Address) : 웹브라우저가 설치된 컴퓨터를 나타내는 주소
파일로 연것 VS 브라우저를 연 것 : 경로가 다르다.
파일로 열기 (file://) : Web Browser -> index.html
주소로 열기 (http:// <-- Wb와 Ws의 통신규약인 hypertext transfer protocol을 이용한다는 의미) : Web Browser -> Web server index.html -> Web Browser.
: 다른 컴퓨터로 Wb와 Ws가 분리되어도 사용이 가능
